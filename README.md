# 水果電商供應商與銷售數據分析 (supplier-sales-analysis)

## 📌 專案概述
本專案為**自主進修**期間之實戰成果。運用模擬的水果電商數據，針對「客戶行為表現與消費生命週期」進行多維度數據分析。專案核心在於運用**SQL**進行精準的資料清洗與聚合，並透過**Power BI**與**Excel**建立結構化指標，以提供營運端改善成本與稽核風險的參考依據。

* **資料庫來源**：臺灣大學資訊系統訓練班線上課程（模擬水果電商資料庫）

---

## 🗄️ 資料庫架構 (Database Schema)
本專案使用 `HappyFruit` 資料庫，包含以下五張主要資料表：

*   **`HappyFruit.dbo.Clients`**：客戶資料主檔。
*   **`HappyFruit.dbo.Employees`**：員工資料主檔。
*   **`HappyFruit.dbo.Products`**：產品資料主檔。
*   **`HappyFruit.dbo.Sales`**：標準銷售交易明細表（以編號紀錄）。
*   **`HappyFruit.dbo.salesall`**：標準銷售交易明細表（以名稱紀錄）。

## 🔍 SQL 商業分析核心案例 (SQL Analysis Cases)
本專案不單純進行數據查詢，著重於在複雜資料環境下的**數據監控**與**商業邏輯嚴謹性**。核心腳本包含：

### 01.產品資料驗證 (`EXISTS` / 子查詢)
*   **技術核心**：運用`EXISTS`子查詢進行跨資料表主檔清洗。
*   **商業價值**：精準篩選歷史流通品項，主動排除未曾發生交易之測試資料與流產品項，確保大盤分析的基礎資料品質。

### 02.核心客戶貢獻度 (`JOIN` / `GROUP BY` / `HAVING`)
*   **技術核心**：多表連結、聚合函數與門檻過濾。
*   **商業價值**：識別高價值客戶，建立業績基準線，用以評估企業是否存在營收過度集中的潛在風險。

### 03.2019年度每月營收匯總與日期預處理 (`REPLACE`)
*   **技術核心**：非標準日期欄位之字串清洗與時間維度聚合。
*   **商業價值**：修正前端系統產出的髒資料，以時間維度做指標監控行銷淡旺季的趨勢變化。

### 04.員工歷史累積業績達標分析 (`GROUP BY` / `HAVING`)
*   **技術核心**：歷史總額聚合累積貢獻篩選。
*   **商業價值**：識別組織內部的歷史核心資產與高貢獻銷售戰力，提供客觀依據，用以輔助內部知識轉移與經驗複製。

### 05.客戶生命週期動態追蹤 (`Window Function` / 視窗函數進階應用)
*   **技術核心**：`LAG()`、`FIRST_VALUE()`、`AVG() OVER (ROWS BETWEEN...)`
*   **商業價值**：在不改變原始訂單資料顆粒度的架構下，同步計算單一客戶的「歷史首購基準」、「前次交易差額」與「三筆訂單移動平均線」，建立動態流失早期預警機制與訂單價值分級流程。
